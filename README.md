# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

В приложении используются два основных типа данных, описываемых следующими интерфейсами.

#### Интерфейс IProduct (Товар)

Описывает структуру данных товара в системе. Каждый товар имеет уникальный идентификатор, описание, изображение, название, категорию и цену (которая может быть null, если товар временно недоступен для заказа).

```
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```

#### Интерфейс IBuyer (Покупатель)

Описывает данные покупателя, необходимые для оформления заказа. Включает способ оплаты (cash или card), email, телефон и адрес доставки.

```
type TPayment = 'cash' | 'card' | '';

interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}
```

### Модели данных

#### Класс Catalog (Каталог товаров)

Отвечает за хранение и управление товарами, доступными для покупки в приложении. Класс хранит полный список товаров и текущий товар, выбранный для детального просмотра.

Конструктор:
`constructor()` - не принимает параметров, создает экземпляр каталога с пустым списком товаров и выбранным товаром `null`

Приватные поля класса:
`items: IProduct[]` - хранит массив всех товаров каталога
`selectedItem: IProduct | null` - хранит товар, выбранный для подробного отображения, или null, если товар не выбран

Методы:
`setItems(items: IProduct[]): void` - сохраняет переданный массив товаров в модель
`getItems(): IProduct[]` - возвращает массив всех товаров из модели
`getItemById(id: string): IProduct | undefined` - возвращает товар по его идентификатору или undefined, если товар не найден
`setSelectedItem(product: IProduct | null): void` - сохраняет товар для подробного отображения
`getSelectedItem(): IProduct | null` - возвращает товар, выбранный для подробного отображения

#### Класс Basket (Корзина товаров)

Управляет товарами, выбранными пользователем для покупки. Обеспечивает добавление, удаление, подсчет стоимости и другие операции с товарами в корзине.

Конструктор:
`constructor()` - не принимает параметров, создает пустую корзину

Приватные поля класса:
`items: IProduct[]` - хранит массив товаров в корзине

Методы:
`getItems(): IProduct[]` - возвращает массив товаров, находящихся в корзине
`addItem(item: IProduct): void` - добавляет товар в корзину
`removeItem(item: IProduct): void` - удаляет товар из корзины
`clear(): void` - полностью очищает корзину
`getTotal(): number` - возвращает общую стоимость всех товаров в корзине
`getCount(): number` - возвращает количество товаров в корзине
`hasItem(id: string): boolean` - проверяет наличие товара в корзине по его идентификатору

#### Класс Buyer (Покупатель)

Хранит и управляет данными покупателя, необходимыми для оформления заказа. Обеспечивает валидацию введенных данных.

Конструктор:
`constructor()` - не принимает параметров

Приватные поля класса:
`payment: TPayment` - хранит способ оплаты
`email: string` - хранит email покупателя
`phone: string` - хранит телефон покупателя
`address: string` - хранит адрес покупателя

Методы:
`setData(data: Partial<IBuyer>): void` - сохраняет данные покупателя. Метод позволяет сохранять как все поля сразу, так и отдельные поля, не затрагивая остальные
`getData(): IBuyer` - возвращает все данные покупателя
`clear(): void` - очищает все данные покупателя
`validate(): ValidationResult` - выполняет валидацию данных покупателя

Тип ValidationResult:
```
type ValidationResult = {
  payment?: string;
  email?: string;
  phone?: string;
  address?: string;
}
```

Метод `validate()` возвращает объект, в котором присутствуют только те поля, которые содержат ошибки валидации. Если поле прошло валидацию, оно отсутствует в возвращаемом объекте.

Правила валидации:
* Поле считается валидным, если оно не пустое
* Для поля payment допустимы значения 'cash', 'card' или ''
* Поле email должно содержать email-адрес
* Поле phone должно содержать номер телефона
* Поле address не должно быть пустым

### Слой коммуникации

#### Класс Communication

Класс отвечает за всю коммуникацию с сервером.

Конструктор:
`constructor(api: IApi)` - принимает в конструктор объект, реализующий интерфейс IApi

Приватные поля класса:
`api: IApi` - базовый API-клиент для выполнения HTTP-запросов

Методы:
`getProductList(): Promise<IProduct[]>` - выполняет GET-запрос к эндпоинту /product/ и возвращает промис с массивом товаров
`postOrder(orderData: IOrder): Promise<IOrderResult>` - выполняет POST-запрос к эндпоинту /order/, принимает данные заказа (покупатель + товары) и возвращает результат оформления заказа

### Слой представления (View)

#### Класс Modal (Модальное окно)

Отображает содержимое в модальном окне и управляет его открытием и закрытием. При открытии и закрытии уведомляет брокер событий.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — принимает DOM-элемент модального окна и объект событий для эмита `modal:open` и `modal:close`.

Защищённые поля класса:  
`closeButton: HTMLButtonElement` — кнопка закрытия  
`contentElement: HTMLElement` — контейнер для подставляемого содержимого  
`events: IEvents` — брокер событий

Сеттеры:  
`content` — подставляет переданный `HTMLElement` в область содержимого (`replaceChildren`).

Методы:  
`open(): void` — показывает модальное окно (класс `modal_active`), эмитит событие `modal:open`  
`close(): void` — скрывает окно, очищает содержимое, эмитит `modal:close`  
`render(data: IModal): HTMLElement` — вызывает базовый `render`, открывает окно и возвращает корневой элемент

#### Класс Header (Шапка)

Отображает счётчик товаров в корзине и при клике по иконке корзины инициирует открытие корзины.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — брокер событий и корневой DOM-элемент шапки.

Защищённые поля класса:  
`basketButton: HTMLButtonElement` — кнопка корзины  
`counterElement: HTMLElement` — элемент со значением счётчика  
`events: IEvents` — брокер событий

Сеттеры:  
`counter` — выводит число товаров в корзине в текстовом виде.

#### Класс Gallery (Галерея каталога)

Контейнер для списка карточек товаров: подставляет массив готовых DOM-элементов в разметку галереи.

Конструктор:  
`constructor(container: HTMLElement)` — корневой элемент галереи (в него же монтируется список).

Защищённые поля класса:  
`catalogElement: HTMLElement` — контейнер списка карточек

Сеттеры:  
`catalog` — заменяет дочерние элементы контейнера переданным массивом узлов (`replaceChildren`).

#### Класс BasketView (Корзина)

Отображает список товаров в корзине, итоговую сумму и кнопку оформления заказа.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — корневой DOM-элемент корзины и опционально объект с колбэком `onClick` для кнопки оформления.

Защищённые поля класса:  
`listElement: HTMLElement` — список позиций  
`buttonElement: HTMLButtonElement` — кнопка оформления  
`totalElement: HTMLElement` — отображение итоговой суммы

Сеттеры:  
`items` — подставляет массив DOM-элементов строк корзины  
`total` — выводит сумму в формате «N синапсов»  
`selected` — блокирует кнопку оформления, если переданный массив идентификаторов пуст

#### Класс Success (Успешное оформление заказа)

Показывает сообщение о списании синапсов и при необходимости обрабатывает закрытие окна.

Конструктор:  
`constructor(container: HTMLElement, actions?: { onClick: () => void })` — корневой элемент и опционально обработчик клика по кнопке закрытия.

Защищённые поля класса:  
`closeButton: HTMLButtonElement` — кнопка закрытия  
`descriptionElement: HTMLElement` — текст описания (списание)

Сеттеры:  
`total` — задаёт текст вида «Списано N синапсов».

#### Класс Form (Базовая форма)

Родитель для форм. Отвечает за показ ошибок и блокировку submit-кнопки.

Конструктор:  
`constructor(container: HTMLElement)` — корневой элемент формы.

Защищённые поля класса:  
`submitButton: HTMLButtonElement` — кнопка отправки (`button[type="submit"]`)  
`errorsElement: HTMLElement` — контейнер текста ошибок (`.form__errors`)

Сеттеры:  
`errors` — выводит текст ошибок  
`valid` — при `false` отключает кнопку отправки, при `true` включает

#### Класс FormOrder (Форма заказа: оплата и адрес)

Форма выбора способа оплаты (карта / наличные), ввода адреса и отправки заказа; синхронизирует состояние с моделью через события.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — корневой элемент формы и брокер событий.

Защищённые поля класса:  
`cardButton`, `cashButton: HTMLButtonElement` — переключатели способа оплаты  
`addressInput: HTMLInputElement` — поле адреса  
`events: IEvents` — брокер событий

Сеттеры:  
`payment` — подсвечивает выбранный способ оплаты (класс `button_alt-active`)  
`address` — устанавливает значение поля адреса  
`errors`, `valid` — делегируют базовому классу `Form`

#### Класс Card (Базовая карточка товара)

Родительский класс для карточек: отображает название и цену. Тип данных карточки задаётся дженериком `T`.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — корневой элемент карточки; `actions` зарезервирован для наследников (базовый класс сам не вешает обработчики).

Защищённые поля класса:  
`titleElement` - название товара
`priceElement: HTMLElement` — цена товара

Сеттеры:  
`title` — текст названия  
`price` — для `null` выводится «Бесценно», иначе строка «N синапсов»


#### Класс CardCatalog (Карточка в каталоге)

Карточка товара в списке каталога: изображение, категория с оформлением по словарю категорий, клик по карточке.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — при переданном `actions.onClick` клик по контейнеру карточки вызывает колбэк.

Защищённые поля класса:  
`imageElement: HTMLImageElement` — изображение товара  
`categoryElement: HTMLElement` — бейдж категории

Сеттеры:  
`category` — текст категории и переключение CSS-классов по `categoryMap`  
`image` — задаёт `src` и `alt` изображения через `setImage` (в качестве `alt` используется текущий `title` из базового класса)

Наследует сеттеры `title` и `price` от `Card`.

#### Класс CardPreview (Карточка предпросмотра товара)

Развёрнутая карточка товара: изображение, категория, описание, цена и кнопка добавления в корзину; при `price === null` кнопка неактивна.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — при `actions.onClick` обработчик вешается на кнопку карточки.

Защищённые поля класса:  
`imageElement`, `categoryElement`, `descriptionElement: HTMLElement` — контент карточки  
`buttonElement: HTMLButtonElement` — кнопка действия

Сеттеры:  
`category`, `image`, `description` — как в каталоге/базовом классе  
`price` — вызывает базовый сеттер цены и отключает кнопку, если цена `null`

#### Класс CardBasket (Строка товара в корзине)

Отображает порядковый номер, название, цену и кнопку удаления позиции.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — при `actions.onClick` обработчик вешается на кнопку удаления.

Защищённые поля класса:  
`indexElement: HTMLElement` — номер позиции  
`deleteButton: HTMLButtonElement` — удаление строки

Сеттеры:  
`index` — порядковый номер  
`title`, `price` — делегируют базовому классу `Card`

### События приложения

Ниже перечислены события, которые генерируются в слое представления при действиях пользователя и при изменении состояния интерфейса:

`basket:open` — клик по кнопке корзины в `Header`.
`modal:open` — открытие модального окна в `Modal`.
`modal:close` — закрытие модального окна в `Modal` (кнопка закрытия или клик по оверлею).
`order:payment` — выбор способа оплаты в `FormOrder`; передаются данные `{ payment: 'card' | 'cash' }`.
`order:change` — ввод адреса в `FormOrder`; передаются данные `{ address: string }`.
`order:submit` — отправка формы заказа в `FormOrder`; передаются данные `{ payment: TPayment, address: string }`.
`contacts:change` — изменение email или телефона в `FormContacts`; передаются частичные данные `{ email?: string, phone?: string }`.
`contacts:submit` — отправка формы контактов в `FormContacts`; передаются данные `{ email: string, phone: string }`.

Также в ряде компонентов (`CardCatalog`, `CardPreview`, `CardBasket`, `BasketView`, `Success`) при действиях пользователя вызывается обработчик `actions.onClick`, полученный в конструкторе. В этом обработчике Презентер должен сгенерировать соответствующее событие через брокер событий.
